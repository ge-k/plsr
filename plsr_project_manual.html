工程学习手册

PLSR 脉冲控制工程的 CubeMX 配置与自定义源码说明

本文件基于当前工程目录生成，只设置两个主章节：先粗略但完整地说明 STM32CubeMX 配置， 再逐个说明非 CubeMX 生成、非 agile_modbus 源码的业务文件。

芯片STM32F103C8T6
主频72 MHz
协议Modbus RTU
输出TIM3 PWM + TIM4 计数

1. STM32CubeMX 的配置

CubeMX 工程文件为 plsr_project.ioc，目标芯片是 STM32F103C8Tx， 封装为 LQFP48，固件包为 STM32Cube FW_F1 V1.8.7，生成工具链选择 MDK-ARM V5.32。工程保留用户代码区，生成入口位于 Core/Src。

时钟树

  • 外部高速晶振 HSE 开启，PLL 源选择 HSE，倍频 x9。
  • SYSCLK、HCLK、APB2 均为 72 MHz。
  • APB1 分频为 36 MHz，APB1 定时器时钟仍为 72 MHz。
  • SysTick 作为 HAL tick，优先级为 15。

工程入口

  • main.c 先执行 HAL_Init、SystemClock_Config。
  • 随后初始化 GPIO、DMA、USART1、TIM3、TIM4、USART3、TIM1。
  • 用户区调用 my_main()，实际业务主循环在自定义文件内完成。

引脚与外设角色

资源配置在工程中的作用
PA6TIM3_CH1，PWM 输出，复用推挽，高速脉冲输出端，当前板级映射中 Y0 至 Y3 都指向该物理脚。
PA7普通推挽输出，默认低电平方向输出端，当前板级映射中 Y12 至 Y15 都指向该物理脚。
PA5上升沿 EXTI，内部下拉外部等待/触发输入，映射为 X4、X5。
PA9 / PA10USART1_TX/RX，异步串口Modbus RTU 通信口，RX/TX 均配置 DMA。
PB10 / PB11USART3_TX/RX，异步串口调试与 VOFA 输出，TX 配置 DMA。
PD0 / PD1HSE 外部晶振系统主时钟来源。
PC14 / PC15LSE 外部晶振保留低速外部时钟资源。
PA13 / PA14SWDIO / SWCLK调试下载接口。

定时器配置

TIM3

TIM3_CH1 配置为 PWM Generation CH1，初始 PSC=0、ARR=65535，主触发输出为 TRGO_UPDATE。业务代码运行时会动态改写 PSC、ARR、CCR1 来生成指定频率和 50% 占空比脉冲。

TIM4

TIM4 使用 ITR2 作为外部时钟触发，等价于对 TIM3 更新事件计数；CH1 配成 Output Compare No Output。业务代码利用 TIM4 CH1 + DMA 在接近目标脉冲数时自动关闭 TIM3。

TIM1

TIM1 配置为基础定时器，PSC=71、ARR=999，更新中断已打开。当前自定义调度主要依赖 HAL_GetTick()，TIM1 在现有业务中没有形成主要控制链路。

DMA 配置

请求DMA 通道方向与数据宽度优先级用途
USART1_RXDMA1_Channel5外设到内存，byte，内存递增Medium接收 Modbus RTU 帧，配合空闲中断。
USART1_TXDMA1_Channel4内存到外设，byte，内存递增Very High发送 Modbus RTU 响应。
USART3_TXDMA1_Channel2内存到外设，byte，内存递增Lowmy_printf 调试输出与 VOFA 数据输出。
TIM4_CH1DMA1_Channel1内存到外设，word，内存不递增Very High比较事件触发 DMA 写 TIM3 CR1，自动停止 PWM。

串口与中断

USART

  • USART1 与 USART3 均为 115200 8N1，无硬件流控，16 倍过采样。
  • USART1 用作 Modbus RTU 从站口。
  • USART3 用作格式化输出口，供调试和 VOFA 曲线显示。

NVIC

  • 开启 DMA1_Channel1/2/4/5、EXTI9_5、TIM1_UP、USART1、USART3 中断。
  • 外设中断优先级基本为 0，SysTick 为 15。
  • EXTI9_5_IRQHandler 调用 HAL EXTI 处理，最终触发自定义 HAL_GPIO_EXTI_Callback。

CubeMX 生成文件边界

Core/、Drivers/、MDK-ARM 下的启动文件、工程文件、HAL/CMSIS 文件和大部分 main.c、gpio.c、dma.c、tim.c、usart.c 都属于 CubeMX/HAL 生成或派生代码。 第二章不逐行展开这些文件，而是聚焦业务侧自定义文件。
